### 🔄 Agent <-> Model Interface Checklist

To make this agent code work, your `model.py` needs to implement two specific methods and handle specific input/output formats.

#### 1. `model.get_initial_percepts(agent)`
* **Called when:** Step 1 of the agent's very first turn.
* **Receives:** The `agent` instance itself.
* **Must Return:** The `percepts` dictionary (format detailed below) based on the agent's starting location.

#### 2. `model.do(agent, action)`
* **Called when:** The end of every agent's turn.
* **Receives:** 
  1. The `agent` instance performing the action.
  2. The `action` dictionary generated by `deliberate()`.
* **Must Do (Side Effects):**
  * Validate if the action is legal.
  * Execute the action on the grid (e.g., move the agent, delete waste, spawn transformed waste).
  * **Crucial:** Update the agent's physical inventory by modifying `agent.wastes` directly (e.g., `agent.wastes["green"] += 1` or `agent.wastes["green"] -= 2`).
* **Must Return:** The updated `percepts` dictionary representing the agent's *new* surroundings after the action is completed.

---

### 📦 Data Formats

#### The `percepts` Dictionary (Returned by Model)
Both `get_initial_percepts` and `do` must return a dictionary in this exact format so the agent's `update` function can parse it:
```python
{
    "current_pos": (x, y),  # The agent's current coordinates (tuple)
    "adjacency_grid": {
        (x, y): [obj1, obj2],       # The cell the agent is currently on
        (x+1, y): [obj3],           # Adjacent cell East
        (x, y+1): [obj4, obj5],     # Adjacent cell North
        # ... include all valid adjacent cells
    }
}
```

#### The `action` Dictionary (Received by Model)
The `action` parameter passed to `model.do` will be exactly **one** of these dictionaries:
* `{"type": "stay"}`
* `{"type": "move", "target": (x, y)}` *(Note: target is a coordinate tuple)*
* `{"type": "pick"}` *(Note: Model must check `agent.type` to know which color to pick from the grid)*
* `{"type": "transform"}`
* `{"type": "put"}`
